# TCP是有连接，面向流，可靠协议
# 有连接是说发送消息之前，要进行建立双向连接
# 面向流：（发送方发送消息到缓存，缓存到一定大小（定期），然后从缓存取出数据发往接收端
# 可能会有粘包现象

# 可靠协议：发送一个消息，都要一个响应。
# 可靠在哪：发送方发送一个消息到缓存，然后协议控制将缓存的数据发往对端，只有接收端返回ack=1 ，
# 发送端则清理缓存中的数据，对端返回ack=0,则重新发送数据（没有将缓存的消息给删除，协议会
重新发送数据），所以TCP是可靠的。


# UDP是无连接，面向消息，不可靠协议

# 面向消息(自带报头，接收方接收的时候，就知道是一条消息），
# 不可靠： 对端不会返回ack，协议控制将缓存数据发送出去之后，就会删除缓存的数据
